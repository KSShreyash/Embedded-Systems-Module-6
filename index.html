I²C

Inter-Integrated Circuit

🧠 Easy Analogy

Think of I²C like a classroom: The teacher (master) calls a student's name (7-bit address) and only that student (slave) responds. Two phone lines only — one to speak (SDA), one to keep rhythm (SCL).

Quick Facts

Invented byPhilips Semiconductor (1989)
Wires2 — SDA + SCL
ModeSynchronous, Serial
Addressing7-bit or 10-bit
TopologyMulti-master, Multi-slave
CommunicationHalf-duplex
ClockControlled by Master

The Two Wires

SDA (Serial Data Line) — Bidirectional. Carries actual data between master and slave.

SCL (Serial Clock Line) — Clock signal, always generated by the master.

Both lines use pull-up resistors. Devices communicate by pulling lines LOW (open-drain logic).

Arduino pins: SDA → A4, SCL → A5
📦 How Data is Transferred

I²C Frame Structure

ACK
8-bit Data

Each message = Address Frame + one or more Data Frames, wrapped with START and STOP conditions.

🔢 6 Steps of I²C Transmission
  • 1START condition: Master pulls SDA LOW while SCL is HIGH — alerts all slaves.
  • 2Address frame: Master sends 7-bit address + R/W bit to identify target slave.
  • 3Address ACK: Matching slave pulls SDA LOW (ACK). Non-matching slaves keep SDA HIGH.
  • 4Data transfer: Master sends (or receives) data frames, 8 bits at a time.
  • 5Data ACK: Receiver sends ACK bit after each successful 8-bit data frame.
  • 6STOP condition: Master pulls SCL HIGH, then SDA HIGH — communication ends.

✅ Advantages

Only 2 wires needed Multi-master & multi-slave support ACK/NACK for error detection Works with slow and fast ICs Chip addressing (no extra CS pin)

❌ Disadvantages

Slower than SPI Pull-up resistors cause power dissipation Hardware complexity in multi-master Frame overhead from ACK/address bits
💡 Memory Trick I²C = "I Communicate" with just 2 wires. SDA = "Say Data Aloud". SCL = "Set Clock Level". Master always controls the clock!
CAN

Controller Area Network

🧠 Easy Analogy

CAN is like a school announcement system: One speaker (bus) reaches every class (ECU). Each announcement has an ID. You listen only if the ID matches yours. No one is "the master" — anyone can speak when the speaker is free!

Quick Facts

Used inAutomobiles, industrial control
Wires2 — CAN_H + CAN_L
ArchitectureMulti-master (no master-slave)
Max Speed1 Mbit/s @ 40m
ID (Standard)11-bit identifier
ID (Extended)29-bit identifier
OSI Layers usedPhysical + Data Link (2 layers)

Why CAN in Cars?

Cars have 70+ ECUs controlling ABS, airbags, AC, gear control, etc. Point-to-point wiring would be a mess!

CAN solves this with one shared bus:

Simple — one bus for all Robust against EM interference Priority-based message access Nodes can be added anytime

📋 4 Types of CAN Frames

Data Frame

Most common. Transmits data from one node to another. Contains: Arbitration + Control + Data + CRC + ACK + EOF fields.

Remote Frame

Requests data from another node. Same as data frame but no data field. RTR bit is recessive (1).

Error Frame

Sent when a node detects an error. Contains Error Flag (dominant bits) + Error Delimiter (8 recessive bits). Aborts current transmission.

Overload Frame

Adds extra delay between messages. Sent when a node is too busy. Looks like error frame but signals "not ready".

🔧 Key Concepts

Bit Stuffing

After 5 consecutive identical bits, an opposite bit is inserted. Ensures clock synchronization and prevents data from looking like a control pattern. Slightly increases message length but ensures integrity.

Example: 11111 → 111110 (stuffed bit added)

Arbitration (Priority)

When multiple nodes transmit, lowest ID wins (dominant bit = 0 wins over recessive 1). Higher priority messages get bus access. Losing nodes automatically retry later.

Non-destructive arbitration = no data lost!

CAN Error Types

Bit Error — bit corrupted on bus Bit Stuffing Error — extra bits detected CRC Error — checksum mismatch ACK Error — no acknowledgement received Form Error — fixed-form field violated

CAN Architecture (OSI)

CAN only uses 2 of 7 OSI layers:

  • Data Link Layer — LLC (message filtering) + MAC (framing, arbitration, ACK)
  • Physical Layer — CAN transceiver handles encoding/decoding on bus
💡 Memory Trick — CAN frame fields "Some Ignore Repetitive Data, Check All Errors Inside Frame" → SOF, ID, RTR, DLC, Data, CRC, ACK, EOF, IFS
ZigBee

ZigBee Protocol

🧠 Easy Analogy

ZigBee is like a relay race for tiny messages: A coordinator starts the race, routers pass the baton, and end devices receive the baton and rest (sleep to save battery). Perfect for IoT sensors that just need to send a small update every now and then!

Quick Facts

StandardIEEE 802.15.4
Frequency2.4 GHz worldwide
Data Rate250 kbps
Range10–100m indoor, 300m outdoor
Network sizeUp to 65,000 nodes
SecurityAES-128 encryption
Network join~30 seconds

Why ZigBee over Bluetooth/WiFi?

Bluetooth and Wi-Fi consume too much power for sensor networks. ZigBee is purpose-built for:

Ultra-low power consumption Large mesh networks (65K nodes) Very low data rate (sensors only need small packets) Low cost per device Ideal for WSN and IoT
🌐 3 Types of ZigBee Devices

Coordinator

  • Root of the network
  • One per network
  • Channel selection
  • Assigns node IDs & addresses
  • Always awake (powered)

Router

  • Intermediate node
  • Routes traffic between nodes
  • Allows end devices to join
  • Always on (powered)
  • Extends network range

End Device

  • Sensor/actuator nodes
  • Can sleep to save battery
  • Traffic routed via parent
  • Battery-operated
  • Simplest device type
🔗 Network Topologies

Channel Access Methods

Contention-Based (CSMA): Device checks if channel is free. If free, transmit. If busy, wait random time then retry. (No synchronization needed)

Contention-Free (GTS): Coordinator allocates Guaranteed Time Slots to devices. Each device gets dedicated time. (Beacon mode)

ZigBee Protocol Stack

  • Application Layer (ZDO + APS)
  • Network Layer (NWK)
  • MAC Layer
  • Physical Layer (IEEE 802.15.4)
💡 Memory Trick ZigBee = "ZIG ZAG paths through a mesh, saving battery like a bee that only visits necessary flowers!" 3 devices: Coordinator (C = Chief), Router (R = Relay), End Device (E = Edge/Energy-saving)
BT

Bluetooth Protocol

🧠 Easy Analogy

Bluetooth is like a rock band with groupies: 1 master (band leader) can control up to 7 slaves (musicians). They all share one stage (2.4 GHz). Multiple bands can share a venue — that's a Scatternet!

Quick Facts

Frequency2.4 GHz ISM band
NetworkPiconet (1M + 7S)
Address48-bit BD_ADDR
ACL max rate721 kbps
SCO rate64 kbps (voice)
Max slaves/piconet7 active, 255 in park mode
TechniqueFHSS — Frequency Hopping

Piconet vs Scatternet

Piconet: 1 master + up to 7 active slaves. Slaves can only talk to the master, not to each other.

Scatternet: Multiple piconets connected — a device acts as both a slave in one piconet and a master in another, bridging two networks.

Each device has unique 48-bit BD_ADDR (like MAC address, shown as 12-digit hex)
📶 Bluetooth Architecture Layers
Applications — User apps (headset, file transfer etc.)
TCS — Telephony control SDP — Service discovery RFCOMM — Serial port emulation
L2CAP — Logical Link Control & Adaptation (multiplexing, segmentation, QoS)
LMP — Link Manager Protocol (authentication, encryption, packet sizes)
Baseband — Packet format, addressing, timing, power control (ACL + SCO links)
Radio — 2.4 GHz, frequency hopping, modulation specs
🔗 Two Link Types (Baseband)

ACL — Asynchronous Connection Less

  • Packet-switched data transfer
  • One ACL link per slave-master pair
  • Used when correctness > speed
  • Max data rate: 721 kbps
  • Used for: file transfer, internet browsing

SCO — Synchronous Connection Oriented

  • Real-time, reserved time slots
  • Damaged packets NOT retransmitted
  • Used when speed > correctness
  • Data rate: 64 kbps (voice quality)
  • Used for: voice calls, headsets
🔄 3-Step Connection Process
  • 1Inquiry: Device broadcasts to discover nearby Bluetooth devices. Responding device sends back its address and name.
  • 2Paging: One device initiates connection to a specific target using its BD_ADDR. This forms the actual link.
  • 3Connection: Device enters one of the connection modes (Active, Sniff, Hold, or Park) based on power/usage needs.
💤 4 Connection Modes
Active Mode — Full send/receive (max 7 slaves) Sniff Mode — Power saving, fixed recurring slot Hold Mode — Sleep for fixed period, then wake Park Mode — Deep sleep, max 255 devices, wake only on master command

Power Classes

Class 1100 mW — 100 m range
Class 22.5 mW — 10 m range
Class 31 mW — 10 cm range
💡 Memory Trick — Connection Modes "A Shark Hunts Prey" → Active, Sniff, Hold, Park (power consumption decreases in that order!)
Wi-Fi

Wi-Fi (IEEE 802.11)

🧠 Easy Analogy

Wi-Fi is like a cell tower for your home: The Access Point (AP) is your base station. Devices connect wirelessly to it, and the AP routes traffic between devices and the internet. No wires needed!

Quick Facts

StandardIEEE 802.11 (WLAN)
Full formWireless Fidelity
AP range (indoor)100–150 feet
AP range (outdoor)300 feet
Max users/AP~30 users
SecurityWPA / WPA2 / WPA3
Network nameSSID (32 chars)

Key Components

  • Access Point (AP) — Base station that connects wireless devices to internet
  • Wi-Fi Card — Internal/external adapter in device to receive/send wireless signals
  • SSID — Network name (up to 32 chars) to identify Wi-Fi network
  • WPA-PSK — Password-based encryption (WPA/WPA2/WPA3)
  • Hotspot — Geographic area with accessible wireless internet
🌐 Two Topologies

Infrastructure Mode (AP-Based)

All communication goes through the Access Point. Device → AP → Destination.

Centrally managed Longer range via multiple APs Internet access through AP Requires AP hardware Single point of failure

Ad-Hoc Mode (Peer-to-Peer)

Devices communicate directly without an AP. Point-to-point network.

No AP needed Quick setup Good for device-to-device sharing Short range only No internet access

✅ Advantages

High mobility — no cables Easy installation and flexibility Uses unlicensed spectrum Supports roaming between APs Reliable and secure (WPA3)

❌ Limitations

Radio interference possible Performance degrades with walls/distance Higher power consumption than ZigBee/BT Limited range without multiple APs
💡 Memory Trick Wi-Fi = "Wire-Free Internet". SSID = "Say Something Identifying Data". AP = Access Point = the wireless "door" to the internet!
SPI

Serial Peripheral Interface

🧠 Easy Analogy

SPI is like a telephone operator console: 4 dedicated lines — one to talk (MOSI), one to listen (MISO), one clock beat (SCLK), and one to "dial" a specific extension (CS/SS). Fully duplex — you can talk and listen at the same time!

Quick Facts

Wires4 — MOSI, MISO, SCLK, CS/SS
ModeSynchronous, Full-Duplex
Masters1 only
SlavesTheoretically unlimited
Max Speed10 Mbps
ACKNo acknowledgement
Data bitsNot limited to 8-bits

4 Signal Lines

  • MOSI Master Out Slave In — Master → Slave data
  • MISO Master In Slave Out — Slave → Master data
  • SCLK Serial Clock — Generated by master only
  • CS/SS Chip Select / Slave Select — LOW = selected
🔄 4 Steps of SPI Communication
  • 1Clock setup: Master generates the clock signal (SCLK). Data bits are synchronized to clock edges.
  • 2Slave selection: Master pulls target slave's CS/SS pin LOW to activate it. All others stay HIGH (inactive).
  • 3Data send (MOSI): Master sends data bit-by-bit on MOSI line. MSB sent first.
  • 4Data receive (MISO): Slave sends response bit-by-bit on MISO line simultaneously. LSB sent first.
⚙️ Two SPI Configurations

Independent Slave Config

Each slave has its own dedicated CS/SS line from master. SCK, MOSI, MISO are shared.

Simple, direct control Each slave independently selected Needs 1 CS pin per slave on master

Daisy Chain Config

Single SS line for all slaves. MISO of slave 1 → MOSI of slave 2 → ... → MISO of last slave → back to master.

Saves CS pins on master Data must pass through all slaves More complex timing
🕐 4 Clock Modes (CPOL + CPHA)
ModeCPOLCPHAClock IdleData Sampled On
Mode 000LOWRising edge
Mode 101LOWFalling edge
Mode 210HIGHFalling edge
Mode 311HIGHRising edge

CPOL = Clock Polarity (idle state). CPHA = Clock Phase (when data is sampled).

✅ Advantages

Full-duplex (simultaneous TX/RX) Very high speed (10 Mbps) No address bits overhead Continuous data streaming Simple slave addressing via CS

❌ Disadvantages

4 wires needed (vs 2 for I²C) Only 1 master allowed No ACK/NACK error confirmation Needs dedicated CS pin per slave
💡 Memory Trick — SPI pins "My Music Sounds Clear" → MOSI, MISO, SCLK, CS. Or remember: Master Outputs (MOSI), Master Inputs (MISO)!
RS-232

RS-232 Serial Interface

🧠 Easy Analogy

RS-232 is like a telephone call: One end is the terminal (DTE — the person speaking), the other is the modem (DCE — the phone exchange). They use a standard set of signals to start, stop, and manage the conversation.

Quick Facts

Full nameRS-232-C (EIA Standard)
Developed byEIA + Bell Systems
ConnectionDTE ↔ DCE
TypeSerial, asynchronous
Voltage (logic 1)-3V to -15V
Voltage (logic 0)+3V to +15V
Max distance~15 meters (50 ft)

DTE vs DCE

DTE (Data Terminal Equipment) — The end device, e.g., a computer, terminal, printer. Generates and consumes data.

DCE (Data Communication Equipment) — The communication device, e.g., a modem. Converts signal for transmission through the telephone network.

DTE → RS-232 interface → DCE → Telephone Network → DCE → RS-232 → DTE
📌 Key RS-232 Signals (DB-9 Connector)
SignalPinDirectionDescription
TXD — Transmit Data3DTE → DCESerial data from DTE to DCE
RXD — Receive Data2DCE → DTESerial data from DCE to DTE
RTS — Request To Send7DTE → DCEDTE wants to send data
CTS — Clear To Send8DCE → DTEDCE ready to receive data
DTR — Data Terminal Ready4DTE → DCEDTE is powered and ready
DSR — Data Set Ready6DCE → DTEDCE is powered and ready
GND — Signal Ground5—Common reference

RS-232 Characteristics

Simple hardware, widely available Good noise immunity with voltage levels Full-duplex (TXD and RXD) Short range (~15m) Slow speed compared to modern protocols Point-to-point only (no bus)

Voltage Logic (Inverted!)

RS-232 uses inverted logic compared to TTL:

Logic 1 (MARK) = -3V to -15V (negative)
Logic 0 (SPACE) = +3V to +15V (positive)

This inversion distinguishes RS-232 from normal TTL/CMOS logic!

💡 Memory Trick — RS-232 DTE = Data Terminal Equipment = "Data creator" (computer). DCE = Data Communication Equipment = "Data carrier" (modem). TXD/RXD = Transmit/Receive Data. RTS/CTS = handshake pair ("Ready To Send?" → "Clear To Send!")
